    A interface pronta.

    O cabo trainer FUTABA possui um curto nele, isso provavelmente diz para o controle que existe um cabo trainrer conectado. Você pode (ainda não testei) precisar conectar um fio onde a linha verde é mostrada, para enganar o transmissor como se realmente existisse um cabo.

    Utilitário de teste da interface para DOS Use este programa para testar sua interface, você pode rodá-lo pelo windows. Ele parecerá não fazer nada até que dê um sinal - será quando ele começará a contar como louco e todos os gráficos de barra devem se tornar unidos aos canais do seu controle.

    Para que o programa de teste funcione, seu transmissor deve estar em modo FM padrão (e NÃO PCM) e sua porta paralela deve ser configurada para usar a Porta de Impressora no IRQ 7 e I/O 378h. Esta é a configuração padrão usada na maioria dos computadores. Também, essa configuração não é a padrão para o FMS, você deve manualmente configurar a porta LPT para 378 dentro das configurações do transmissor do FMS para que ele funcione.

    Se você tiver uma placa de som PCI Soundblaster (e talvez outras placas) pode ser necessario desabilitar a placa no gerenciador de dispositivo. Eu fiz um upgrade de minha placa de audio recente e minha interface mostrava" sinal detectado " mas não mostrava nenhum movimento. Quando eu desabilitei a placa, ai sim funcionou bem. Agradecimentos a Simon que divulgou isto em meu forum .

    Nos transmissores FUTABA, o sinal é o pino superior esquerdo quando estamos olhando o conector do transmissor (Pino 2) e o pino de baixo do meio (no anel do conector) é o terra.
